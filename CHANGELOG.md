# Changelog

## [3.24.0] - 2025-04-12

### Новая структура сайта — Проектная организация

#### Архитектура
- **Модель Project** — проекты для группировки сравнений:
  - Поля: `id`, `user_id`, `name`, `created_at`, `updated_at`
  - Связь: `comparisons = db.relationship('Comparison')`
  - Каждый проект принадлежит пользователю и содержит множество сравнений

- **Модель Comparison** — обновлена для поддержки проектов:
  - Добавлено поле `project_id` (ForeignKey к Project)
  - Сохраняется привязка сравнений к конкретному проекту
  - История сравнений теперь проект-специфична

#### Страницы и интерфейс
- **Главная страница (landing)** — новая страница с:
  - Профессиональным описанием возможностей сайта
  - Демонстрацией функционала
  - Призывом к регистрации

- **Личный кабинет (profile)** — обновленный интерфейс:
  - Список проектов пользователя в виде карточек
  - Кнопка создания нового проекта
  - История сравнений внутри проекта

- **Страница проектов (projects)** — центр управления проектами:
  - Создание проекта с названием
  - Карточки проектов с индикаторами метрик
  - Переход к сравнениям внутри проекта

- **Миграция БД** — созданы таблицы:
  - `project` — таблица проектов
  - Обновлена таблица `comparison` — добавлен `project_id`

### Стартовые метрики проекта — Полная реструктуризация

#### Добавлено
- **Новая структура стартовых метрик** — 5 тональностей для 4 групп (Яндекс/Гугл × ТОП-10/ТОП-20):
  - Домашний сайт (client_site)
  - Позитивная тональность (positive)
  - Нейтральная тональность (neutral)
  - Негативная тональность (negative)
  - Нерелевантная тональность (irrelevant)
  - Тип ввода: проценты (percentage) или количество (count)
  - Всего URL для расчета

- **Поля в модели БД Project** — 28 новых полей:
  - Яндекс ТОП-20: `y20_client_site`, `y20_positive`, `y20_neutral`, `y20_negative`, `y20_irrelevant`, `y20_input_type`, `y20_total_urls`
  - Яндекс ТОП-10: `y10_client_site`, `y10_positive`, `y10_neutral`, `y10_negative`, `y10_irrelevant`, `y10_input_type`, `y10_total_urls`
  - Google ТОП-20: `g20_client_site`, `g20_positive`, `g20_neutral`, `g20_negative`, `g20_irrelevant`, `g20_input_type`, `g20_total_urls`
  - Google ТОП-10: `g10_client_site`, `g10_positive`, `g10_neutral`, `g10_negative`, `g10_irrelevant`, `g10_input_type`, `g10_total_urls`

- **Методы модели Project**:
  - `get_baseline(search_engine, top_n)` — получение метрик для конкретной группы
  - `set_baseline(search_engine, top_n, metrics)` — сохранение метрик
  - `has_baseline_metrics(search_engine, top_n)` — проверка наличия метрик

- **Модальное окно ввода метрик** (`projects.html`):
  - 4 таба для каждой группы (Яндекс/Гугл × ТОП-10/ТОП-20)
  - Выбор типа ввода: проценты или количество
  - Поле "Всего URL"
  - 5 полей тональностей с цветовыми иконками
  - Форма идентична интерфейсу сравнения

- **Автоматическое создание стартовых диаграмм**:
  - 4 диаграммы генерируются при сравнении файлов
  - Отдельные диаграммы для Яндекс ТОП-20, Яндекс ТОП-10, Google ТОП-20, Google ТОП-10
  - Диаграммы отображаются в секции "Стартовые метрики проекта" на странице сравнения

- **Индикатор метрик в проекте**:
  - В карточке проекта отображается иконка 📈 если есть стартовые метрики

- **Миграция БД** (`migrate_db.py`):
  - Добавлены все 28 новых колонок для стартовых метрик

#### Удалено
- Старые поля `yandex_start_positive`, `yandex_start_negative`, `yandex_start_neutral`
- Старые поля `google_start_positive`, `google_start_negative`, `google_start_neutral`
- Удалены устаревшие методы `get_yandex_baseline()`, `get_google_baseline()`

### Файлы изменены
- `app.py` — новая модель Project, поля project_id в Comparison, роуты /project/*, /profile, обновлена логика сравнений
- `templates/index.html` — landing page с описанием возможностей
- `templates/profile.html` — обновленный личный кабинет со списком проектов

- `version.txt` — обновлена до 3.24.0

## [3.21.0] - 2025-04-11

#### Добавлено

  - Создаёт PNG-изображения с цветовой таблицей запросов и их тональности
  - Отдельные сетки для Яндекс и Google
  - Отдельные сетки для Файла 1 и Файла 2 (итого 4 сетки)

- **Поля в модели БД** — добавлены новые поля в таблицу `Comparison`:

  - `grid_g10_file2_path` — путь к сетке Google ТОП-10 Файл 2
  - (поля `grid_y10_path` и `grid_g10_path` уже существовали для Файла 1)

- **Генерация сеток при сравнении** — интеграция в основной процесс сравнения:
  - Вызов `create_top10_grid_png()` после анализа данных

  - Запись путей в базу данных
  - Отдельные сетки для каждого файла с подписями имен файлов

  - Секция "Сетка тональности ТОП-10" с заголовком и иконкой

  - Кнопки скачивания для каждой сетки

- **Отображение в истории** — полная интеграция в `view_comparison.html`:

  - Поддержка всех 4 типов сеток

- **PowerPoint презентации (PPTX)** — генерация слайдов с сетками:
  - Новая функция `create_top10_grid_pptx()` — создаёт PPTX с сетками ТОП-10
  - Если < 7 запросов: один слайд с двумя сетками (file1 сверху, file2 снизу)
  - Если >= 7 запросов: два слайда (отдельно для каждого файла)
  - Отдельные PPTX для Яндекс и Google
  - Кнопки скачивания PPTX на странице сравнения
  - Добавлена зависимость `python-pptx==0.6.23` в requirements.txt

### Доработки и правки в процессе создания

#### Высота строк и масштабирование
- **Множество итераций масштаба** — подбор оптимальной высоты ячеек:
  - Начальное значение: `table.scale(1, 2.5)` — слишком высоко
  - Пробовали: 1.2, 0.8, 0.6 — всё ещё не идеально
  - Финальное решение: динамический масштаб
    - 1-3 запроса: `height_scale = 0.5` (компактно)
    - 4+ запросов: `height_scale = 0.8` (квадратные ячейки)
  - Компенсация увеличения шрифта: `table.scale(1, height_scale * 0.75)`

- **Высота фигуры matplotlib** — адаптация под количество запросов:
  - Начально: `max(6, len(queries) * 0.6 + 2)` — фиксированная
  - Пробовали: `max(4, len(queries) * 0.3 + 1.5)` — слишком низко
  - Финальное: `max(3.5, num_queries * 0.45 * height_scale + 1.5)`

#### Шрифты и читаемость
- **Размер шрифта** — несколько попыток:
  - Начально: 9px — слишком мелко
  - Финальное: 12px — читаемо, но скомпенсировано масштабом

- **Перенос текста** — добавлен `cell.set_text_props(wrap=True, ha='left')` для колонки "Запрос"

#### Цветовая схема
- **Оптимизация цветов** — подбор насыщенности:
  - Домашний: `#F9CB9C` (теплый оранжевый)

- **Легенда** — добавлена вручную через matplotlib patches:
